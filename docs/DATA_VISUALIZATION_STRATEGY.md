# Data Visualization Strategy

## Purpose

Документ задает стратегию развития визуализации данных для premium corporate web-presentation platform.

Его задача — зафиксировать, что платформа должна поддерживать не только базовые доказательные элементы, но и продвинутую аналитику, интерактивные executive-слои и фирменные выразительные визуальные форматы. Первый прототип может быть простым технически, но будущая platform не должна мыслиться минималистичной или примитивной.

## Core Principle

Сложная визуализация допустима, если она:

- усиливает управленческий смысл;
- помогает доказать тезис;
- остается читаемой на desktop;
- имеет осмысленную mobile-версию для iPhone Pro Max;
- не превращает презентацию в generic BI dashboard;
- не является декоративной демонстрацией технологии.

Первый прототип может быть минимальным, но platform не должна мыслиться минималистичной. Сложная визуализация не запрещена и не откладывается навсегда. Она просто не входит в самый первый минимальный implementation step, пока platform проверяет базовую формулу:

```text
тезис -> подтверждение данными -> интерпретация -> управленческий вывод
```

## Levels of Data Visualization

Визуализация данных в platform развивается по уровням зрелости. Каждый следующий уровень добавляет выразительность, глубину или интерактивность, но не отменяет базовый принцип: визуализация должна доказывать управленческий тезис.

### 1. Core Evidence Primitives

Это базовые доказательные элементы для первого слоя реализации:

- KPI strip;
- comparison bars;
- annotated metric;
- ranked list;
- decision block;
- simple funnel.

Они нужны, чтобы быстро доказать тезис: крупные числа, база сравнения, аккуратные дельты, короткая интерпретация и управленческий вывод.

### 2. Advanced Analytical Primitives

Это второй уровень аналитики для более глубоких управленческих разборов:

- multi-axis charts;
- waterfall / bridge;
- heatmap;
- matrix;
- cohort / retention views;
- contribution analysis;
- category mix;
- consultant performance;
- channel efficiency;
- dense but designed tables.

Плотные данные допустимы, если они отредактированы как executive artifact: выделяют главное, объясняют аномалии, показывают базу расчета и не заставляют руководителя самому искать вывод.

### 3. Executive Interactive Layers

Это интерактивный слой, который делает web-презентацию сильнее PowerPoint/PDF, но не превращает ее в BI-систему:

- drill-down;
- appendix mode;
- scenario switching;
- period/channel/category toggles;
- expandable evidence;
- before/after mode.

Интерактивность должна помогать обсуждению: открыть детализацию, сравнить сценарии, сменить период, показать приложение. Она не должна требовать от пользователя исследовать интерфейс, чтобы понять главный смысл.

### 4. Signature Visual / Cinematic Primitives

Это выразительный слой будущей platform, который может сделать презентацию запоминающейся и брендово сильной, если связан с бизнес-смыслом:

- 3D product/data scenes;
- Sankey / network visuals;
- animated flow maps;
- service journey;
- Cybertruck delivery route;
- watercolor-style executive illustrations;
- material data sculptures;
- motion-led transformation charts.

Эти форматы допустимы не как украшение, а как фирменный способ объяснить сложные связи: поток заказов, клиентский путь, доставку, вклад каналов, трансформацию метрик или сервисную модель.

## Admission Criteria

Сложная визуализация допускается в презентацию, если:

- есть ясный управленческий тезис;
- визуализация доказывает или объясняет этот тезис;
- есть база сравнения или контекст;
- есть интерпретация;
- есть управленческий вывод;
- есть desktop-версия;
- есть стратегия для iPhone Pro Max portrait и landscape;
- визуализация не превращается в dashboard widget без смысла.

## First Implementation Layer

Первым слоем реализации должны быть:

- comparison bars;
- annotated metric;
- decision block.

`KpiStrip` уже существует и может быть развит. `simple funnel` лучше добавить после первой секции сравнения, чтобы не смешивать сразу несколько типов доказательной логики.

## Second-Stage Roadmap

Во второй этап стоит заложить:

- ranked list;
- simple funnel;
- waterfall / bridge;
- matrix;
- category mix;
- channel efficiency;
- consultant performance;
- dense but designed tables.

Это даст platform язык для sales review, marketing review, категорий, консультантов и операционных разборов.

## Experimental Signature Layer

В отдельном экспериментальном слое остаются:

- 3D product/data scenes;
- Sankey / network visuals;
- animated flow maps;
- Cybertruck delivery route;
- watercolor-style executive illustrations;
- material data sculptures;
- motion-led transformation charts.

Их нужно внедрять после стабилизации базовой доказательной структуры, чтобы эффектная сцена не появилась раньше управленческой необходимости.

## Mobile Transformation Rules

Сложная визуализация на iPhone не должна просто сжиматься. Она должна получать отдельную mobile-форму.

Возможные формы:

- карточки;
- вертикальная последовательность;
- упрощенный рейтинг;
- раскрываемое приложение;
- короткий вывод перед детализацией.

iPhone Pro Max portrait и landscape должны учитываться отдельно. Если визуализация не может быть осмысленно прочитана на iPhone, ей нужна отдельная mobile-структура или перенос в приложение.

## How To Preserve Premium Quality

1. Один экран — один главный тезис.
2. У каждой визуализации есть база сравнения и управленческий вывод.
3. Amber остается точным signal color, а не декоративным цветом для всего.
4. Плотные данные проходят редактуру: группировка, акценты, подписи, пояснение аномалий.
5. Движение раскрывает данные по этапам, а не демонстрирует технологию.
6. Стиль остается material, editorial, restrained: без SaaS-синего свечения, случайных градиентов и декоративной перегрузки.
7. Сложность допустима, если она выглядит как управленческий инструмент, а не как технический аттракцион.

## Summary

Первый прототип может быть минимальным, но platform не должна мыслиться минималистичной. Мы начинаем с доказательных элементов, чтобы закрепить управленческую ясность, а затем осознанно расширяем язык до сложной аналитики, интерактивности и фирменных визуальных сцен.
