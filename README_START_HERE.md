## Чего намеренно нет внутри

Внутри проекта намеренно нет файла `CODEX_UI_PROMPTS.md`.

Промпты для работы через Codex UI лучше хранить отдельно у пользователя как личную шпаргалку. Они не являются project instructions и не должны смешиваться с файлами, которые Codex читает как правила проекта.

## Как использовать

1. Создайте новый проект/репозиторий в Codex UI.
2. Добавьте эти файлы в корень нового проекта с сохранением структуры папок.
3. Убедитесь, что папка `.agents/skills/` действительно попала в проект. На macOS папки с точкой могут быть скрыты в Finder.
4. Первым сообщением в Codex UI отправьте стартовый промпт вручную из вашей личной шпаргалки или из ответа ChatGPT.
5. Не просите Codex “создать недостающие инструкции”, если вы не хотите, чтобы он сам писал правила для себя.

## Главная идея

Старая стабильная web-презентация используется только как visual reference. Новый проект не должен быть заперт в ее архитектуре, количестве слайдов, старой навигации или plain HTML/CSS/JS.

Мы фиксируем brand DNA, но разрешаем креатив в форме, motion, инфографике и интерактивности.
