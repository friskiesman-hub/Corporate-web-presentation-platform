# REFERENCE_USAGE.md

Документ описывает, как использовать предыдущую стабильную web-презентацию BORK Internet Boutique Q2 2026 в новом проекте.

## 1. Статус предыдущей презентации

Предыдущая презентация — это visual reference и stable demo.

Она показывает:

- найденное ощущение премиальности;
- подход к цвету;
- работу с интерьерным/брендовым настроением;
- примеры плавной подачи бизнес-результатов;
- важность mobile/iPhone проверки;
- то, что web-презентация может выглядеть сильнее обычного PowerPoint.

Она не является:

- техническим шаблоном;
- обязательной архитектурой;
- ограничением по количеству секций;
- запретом на React/TypeScript/Vite;
- запретом на новые анимации;
- запретом на новые layout-паттерны;
- финальной дизайн-системой.

## 2. Что можно переиспользовать

Можно переиспользовать или переосмыслить:

- logo;
- pattern;
- crest / герб, если доступен;
- цветовую палитру;
- принципы premium spacing;
- ощущения теплого графита, камня, дерева, ivory, amber;
- кинематографичность;
- идею business review как digital experience;
- lessons learned по iPhone Safari.

## 3. Что нельзя копировать как ограничение

Нельзя автоматически копировать:

- ровно 5 слайдов;
- old slide structure;
- old CSS classes;
- old JS navigation;
- old layout logic;
- plain HTML/CSS/JS как обязательное решение;
- старый уровень анимационной сдержанности;
- старую QA-логику на 5 sections.

## 4. Как прикладывать reference в Codex UI

Рекомендуемый способ:

1. Приложить архив старой стабильной презентации или screenshots.
2. Приложить brand assets отдельно, если есть.
3. В промпте явно написать:
   - использовать как visual reference;
   - не использовать как архитектурное ограничение;
   - сохранить brand DNA;
   - разрешить новые композиции и motion.

## 5. Формулировка для Codex

Используй такую формулировку:

```text
Предыдущая стабильная презентация приложена как visual reference. Она показывает направление премиальности, цветовую атмосферу и brand feel. Не воспринимай ее как техническое ограничение: не фиксируй 5 слайдов, старую структуру, plain HTML/CSS/JS, навигацию или layout. В новом проекте можно использовать современный стек и более смелые анимации, сохраняя brand DNA.
```

## 6. Критерий правильного использования reference

Правильно:

- новый экран ощущается как развитие бренда;
- есть связь через цвет, тон, материалы, знак, паттерн;
- композиция и motion могут быть новыми;
- результат современнее и масштабируемее старой версии.

Неправильно:

- Codex просто копирует старую структуру;
- новая платформа превращается в “ещё одну 5-слайдовую HTML-презентацию”;
- креатив подавлен страхом сломать старую демо-версию.
