# Corporate Web Presentation Skill

## Purpose

Создавать премиальные корпоративные web-презентации, которые соединяют business storytelling, визуальный лоск, современную анимацию, адаптивность и data-driven инфографику.

Главная цель — не один красивый экран, а повторяемый формат, который позже может стать платформой генерации фирменных презентаций на основе бизнес-данных.

## Когда использовать skill

Используй для:

- executive business reviews;
- Internet Boutique performance presentations;
- marketing / promotion reviews;
- sales reviews;
- e-commerce analytics presentations;
- animated infographics;
- reusable presentation templates;
- section architecture;
- web-storytelling;
- premium presentation UI.

Не используй для:

- обычных сайтов;
- e-commerce checkout/product pages;
- raw dashboards;
- generic landing pages;
- unrelated application features.

## Core principle

Сначала история. Потом экран. Потом motion.

Не начинай с компонентов, пока не понятны:

- audience;
- business context;
- core thesis;
- decision goal;
- data basis;
- management implication.

## Workflow

### 1. Понять бизнес-задачу

Определи:

- кто аудитория;
- какая тема;
- какой период;
- какой результат обсуждается;
- какой вопрос должны решить руководители;
- какие данные доступны;
- какие данные отсутствуют;
- что является фактом, а что гипотезой.

### 2. Собрать narrative

Предложи структуру:

1. Executive thesis
2. Context
3. Key result
4. What changed
5. Why it changed
6. Risks / constraints
7. Opportunities
8. Recommendation / next focus
9. Appendix if needed

Не добавляй секцию только потому, что есть данные. Секция нужна, если она продвигает управленческую историю.

### 3. Выбрать section patterns

Возможные patterns:

- ExecutivePerformanceHero
- KPI Summary
- Commercial Result
- Channel Efficiency
- Conversion Funnel
- Category Mix
- Marketing Impact
- Consultant Performance
- Client Journey
- Operational Risk Map
- Scenario Matrix
- Timeline / Roadmap
- Decision Summary
- Appendix Data Layer

### 4. Определить desktop/mobile поведение

Для каждой секции опиши:

- desktop layout;
- iPhone Pro Max portrait layout;
- iPhone Pro Max landscape layout;
- что происходит с chart на mobile;
- где navigation;
- как работает motion.

Секция, которая хорошо выглядит только на desktop, не считается готовой.

### 5. Использовать 21st.dev Magic MCP избирательно

21st.dev можно использовать только как ускоритель UI-вариантов.

Хорошие запросы:

- Generate premium KPI card variations for an executive web presentation.
- Create a responsive timeline for a business roadmap.
- Create a cinematic hero section with restrained but modern motion.
- Create a comparison matrix for channel performance.

Плохие запросы:

- Make the whole presentation.
- Make it luxury.
- Design the brand.
- Create a random dashboard.

После генерации:

1. проверь на generic SaaS;
2. убери лишний glow/gradient/noise;
3. адаптируй к brand tokens;
4. усили типографику;
5. проверь mobile;
6. сохрани только reusable решения.

### 6. Применить premium review

Проверь:

- выглядит ли дорого;
- считывается ли главный тезис за 3 секунды;
- нет ли dashboard clutter;
- не выглядит ли как AI-generated UI;
- достаточно ли смелости по сравнению со stable reference;
- сохранен ли brand DNA;
- работает ли mobile;
- помогает ли секция управленческому решению.

## Data rules

Для каждой ключевой метрики уточняй:

- period;
- base;
- numerator;
- denominator;
- exclusions;
- comparison period;
- create date / complete date / другой принцип расчета.

Не превращай таблицу в слайд. Найди insight и преврати его в тезис.

## Output standard

Для концепции дай:

- title;
- business purpose;
- audience;
- narrative structure;
- section list;
- dominant message per section;
- visual pattern;
- data needed;
- desktop behavior;
- iPhone behavior;
- risks;
- implementation plan.

Для реализации дай:

- changed files;
- component/data contract;
- verification steps;
- mobile risks;
- what became reusable.

## Done criteria

Задача готова, если:

- история coherent;
- brand DNA сохранен;
- visual quality premium;
- desktop работает;
- iPhone Pro Max продуман;
- motion помогает смыслу;
- данные объяснены;
- component reusable where appropriate;
- результат не похож на generic SaaS dashboard.
